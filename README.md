# 北疆红韵 —— 东北红色精神思政资源整合平台

> **大学生创新创业训练计划（大创）项目申报作品**

## 1. 项目定位
本项目旨在利用数字化技术，深度整合黑龙江省及东北地区的红色思政资源，构建一个集成化、交互式、全天候的红色文化传承平台。聚焦“龙江精神”与“东北四大精神”（抗联精神、北大荒精神、大庆精神、铁人精神），为高校师生提供一站式、轻量化的科普与学习服务。

## 2. 核心功能
- **九大一级导航**：首页、东北四大精神、龙江精神、思政图文、视频资源、教学素材、文献资料、科普园地、项目介绍。
- **海量素材填充**：分类展示理论文章、学习推文、红色故事、思政微课、红色纪录片、PPT课件、历史文献等。
- **深度专题体验**：针对核心精神设立独立详情页，包含精神内涵、历史溯源、代表人物等板块。
- **智能化交互**：支持分类筛选、导航悬浮、返回顶部、移动端自适应等现代Web交互功能。

## 3. 技术栈
- **前端核心**：HTML5, CSS3 (Flexbox/Grid), Vanilla JavaScript
- **图标/UI**：FontAwesome 6.0, Source Han Serif SC (思源宋体)
- **适配方案**：完全响应式设计，适配手机、平板及PC端。

## 4. 云端部署指南

## 4.1 静态海报资源维护
- 当前人物专题图、专题视频封面、历史场景封面已统一收口到 `assets/posters/` 目录，适合直接覆盖上传到云端。
- 数据入口统一位于 `js/portal-data.js`，浏览器默认读取 `assets/posters/*.svg` 静态路径。
- `js/portal-data.js` 顶层新增 `assetSourceRegistry`，用于登记核心人物/地标的官方素材来源、推荐图型与替换备注。
- 已确认来源的卡片条目新增 `sourceHintKey`，后续替换正式原图时可按键值一一对应，避免人物串图、场景错配。
- 如需在调整文案后重新生成全部专题图，可在项目根目录执行：

```bash
node ./tools/generate-posters.mjs
```

- 如需替换为最终官方实拍原图，优先直接替换 `portal-data.js` 对应条目的 `image` 路径，或以同名静态文件方式覆盖 `assets/posters/` 中对应资源。
- 推荐替换流程：
  1. 先在 `assetSourceRegistry` 中找到对应 `sourceHintKey` 的来源页。
  2. 从权威站点下载或申请使用原图，统一裁切后放入 `assets/posters/` 或新建 `assets/official/`。
  3. 将对应条目的 `image` 改为正式图片路径，保留 `sourceHintKey` 便于后续追溯。
  4. 替换后检查 PC 与移动端卡片裁切效果，确认未破坏现有政务化布局。
- 当前静态海报仅作为“准确卡位版”过渡素材，目的在于先彻底消除人物串图、封面乱用和场景错配问题。

为了实现任意浏览器直接访问，建议将本项目部署至以下平台：

### 方案 A：Gitee Pages (国内首选，访问速度快)
1. 在 Gitee 上创建一个新仓库，命名为 `beijiang-red-soul`。
2. 将本地代码上传至仓库。
3. 在仓库页面点击 **服务 -> Gitee Pages**。
4. 选择部署分支（通常为 `master`），点击“启动”。
5. 启动成功后，即可获得一个形如 `https://yourname.gitee.io/beijiang-red-soul` 的永久访问链接。

### 方案 B：GitHub Pages (国际通用)
1. 在 GitHub 上创建仓库并上传代码。
2. 进入 **Settings -> Pages**。
3. 在 **Branch** 下选择 `main` 分支和 `/ (root)` 文件夹，保存。
4. 几分钟后，项目将部署至 `https://yourname.github.io/beijiang-red-soul`。

### 方案 C：Vercel / Netlify (自动化程度高)
1. 关联 GitHub 账号。
2. 导入本项目仓库。
3. 点击 **Deploy**，系统会自动生成一个随机二级域名链接。
